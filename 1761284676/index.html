Linux SSH 封禁多次登录失败 IP

2025-10-24 | Linux
条评论

背景

服务器只要对外暴露端口，就会引来各种扫描。今天登录服务器，查看 /var/log/auth.log 日志，发现有几乎一直有人尝试用 ssh 登录服务器。虽然没有成功的，但是让人一直暴力破解也有些安全隐患。 上网查了一些方案，基本上都是通过修改 /etc/hosts.deny 来封禁 IP。 本文也记录一下本次操作的方法，便于以后再有新的服务器，也能快速做自动封禁。

封禁 IP

创建一个脚本 /root/scripts/ban_auth_failed_ip.sh ，读取日志 /var/log/auth.log，提取登录失败次数过多的 IP。这里做了去重，只有被封禁的 IP 不在 /etc/hosts.deny 的时候才新增封禁。 脚本里把阈值设定为 6 次了，如果有需要可以自行修改。

解封 IP

防止封禁的 IP 列表过长，创建一个脚本 /root/scripts/recover_host_deny.sh，自动清空 /etc/hosts.deny

同时为了防止把自己的 IP 封禁了，可以修改 /etc/hosts.allow，把常用的 IP 添加进去。hosts.allow 的优先级是高于 hosts.deny 的。

定时任务

为了实现自动封禁和解封，可以使用 crontab 设置定时任务，自动执行脚本。由于脚本需要有权限修改 /etc/hosts.deny 文件，可以设置 root 的 crontab，执行 crontab -e

设置每隔 2 分钟自动执行 /root/scripts/ban_auth_failed_ip.sh 封禁 IP。

每天的 23:01 分，自动执行 /root/scripts/recover_host_deny.sh 解封全部 IP。
